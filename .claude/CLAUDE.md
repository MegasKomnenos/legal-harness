# Rules

- 모든 내부 사고(thinking)를 한국어로 진행할 것.

## 법률 문서 작업 시 핵심 제약

법률 문서 생성 작업(`/legal-harness` 스킬)에서는 다음 원칙이 적용된다. 스킬이 로드되지 않은 일반 세션에서도 이 디렉토리의 법률 관련 파일을 다룰 때는 아래 원칙을 준수한다.

- **서브에이전트 금지**: 법률 문서의 탐색·읽기·생성 과정에서 서브에이전트(Explore, Agent 등)를 사용하지 않는다. 메인 에이전트가 Read/Bash 도구로 직접 수행한다.
- **전문 전체 적재 원칙**: 판례, 법령, 사건 문서를 읽을 때 snippet이나 부분 검색으로 대체하지 않고 전문 전체를 Read 도구로 읽어 메인 컨텍스트에 직접 적재한다.
- **판례 전문 통독 원칙**: 판례를 인용하려면 `법령_판례/판례/`에서 해당 판례 PDF를 `python3 extract_all.py`로 추출하여 전체를 읽는다. 인용 사전의 요약만으로는 부족하다.

## 불변 자산 보호

- `harness/` 디렉토리의 파일을 직접 수정하지 않는다. 수정이 필요하면 사용자에게 확인한다. 수정 시에는 `harness/10_유지보수_가이드.md`의 절차를 따른다.
- `법령_판례/` 디렉토리의 원본 법령·판례 파일을 수정·삭제하지 않는다.

## 작문 규칙

- em dash(—)를 사용하지 않는다.
- 과잉 설명, 같은 말 반복, 불필요한 재진술을 하지 않는다.
