# 대조표 양식 (adapt-skill 4단계)

지시 단위 대조표의 확정 스키마. 4단계에서 임시 작업 위치에 작성하고, 7단계에서 생성물의 `references/원본_대조.md`로 확정 배치한다. 이 표가 커스텀본의 "왜 이렇게 다른가"를 소급 설명하는 유일한 기록이므로, 원문 인용과 근거를 아끼지 않는다.

## 파일 구조

```markdown
# 원본 대조: {플러그인}:{스킬명}

- 원본: {마켓플레이스}/{플러그인}/{버전}, 경로 {경로}
- 원본 SKILL.md 해시: {git hash-object 값}
- 대조 일자: {YYYY-MM-DD}
- 대조한 로컬 정본: {목록}
- 독립 검증: {수행 일자, 반영 요지}

## 대조표
{아래 10열 표}

## 절 귀속 확인
{원본의 절(heading) 목록 대 행 귀속. "지시 없음(순수 설명)" 판정 절 명시}

## 판정 분포
{축별 행 수. 전 행 무충돌이었다면 재점검 수행 여부}
```

## 10열 스키마

| 열 | 내용 | 작성 요령 |
|---|---|---|
| ID | T-01, T-02, ... | 원본 등장 순서대로 |
| 원본 위치 | 절 제목(과 필요시 행 범위) | 소급 대조 가능하게 |
| 지시 원문 | **인용** | 요약하지 않는다. 요약은 표상 드리프트를 만든다. 길면 핵심 문장만 인용하고 생략 표시 |
| 유형 | 명령 / 금지 / 전제 / 기본값 / 톤 | 복수 가능 |
| 원본 의도 | 이 지시가 지키려는 가치 한 문장 | **이 열을 채우기 전에는 처리안을 낼 수 없다** |
| 대조 로컬 규범 | 층위 + 파일 + 절 | 무충돌이어도 "무엇과 대조했는지" 적는다. 관련 규범이 없으면 "전 층위 관련 규범 부재" |
| 판정 축 | A~F | SKILL.md의 6축 |
| 처리안 | 유지 / 삭제 / 대체 / 조건화 / 추가 / 개정제안 | 개정제안은 커스텀본에 넣지 않고 별도 보고 |
| 대체·조건 수단 | 대체·조건화·추가일 때 그 구체 수단 | "로컬 정본 X를 참조" 식으로. 규범 본문을 복제하지 않는다 |
| 근거 | 판정의 이유 | 빈 칸 금지. 특히 무충돌 행 |

## 대화 제시용 6열 요약

6단계 보고에서 대화에 펼치는 축약형: ID / 지시 요지 / 원본 의도 / 판정 축 / 처리안 / 근거. 전체 10열 표는 파일 경로로 첨부한다.

## 작성 예시 행

| ID | 원본 위치 | 지시 원문 | 유형 | 원본 의도 | 대조 로컬 규범 | 축 | 처리안 | 수단 | 근거 |
|---|---|---|---|---|---|---|---|---|---|
| T-03 | The Iron Law | "NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST. Write code before the test? Delete it." | 명령·금지 | 산출 전에 실패 조건을 정의해, 생성자가 자기 산출을 사후 합리화로 통과시키는 자기기만을 차단 | 절차 정본: legal-harness 8단계(self-check는 기계 선별로 한정)·9단계(독립 검증 게이트) | A 전제 불일치 | 대체 | 산문 산출물에는 실패 테스트가 없으므로, "생성 전 검증 기준 선행 확정 + 생성 후 독립 검증 게이트"로 같은 의도를 달성. 로컬 검증 루프 정본을 참조 | 테스트 스위트 부재(전제 불일치)이나 의도(자기기만 차단)는 로컬 장치가 이미 담당. 절대성(예외는 사용자 허가)은 의도적 설계이므로 대체문에도 보존 |
| T-11 | Support | "Thank your human partner" 금지 등 톤 규범 | 톤 | 아첨성 동조가 검증 없는 수용으로 이어지는 것을 차단 | advisory: CLAUDE.md 작문 규칙(과잉 표현 억제) | D 관례 불일치 | 대체 | 한국어 산문 관례로 변환하되 "검증 없는 동조 금지"의 강도는 유지 | 표층(영어 표현 목록)은 로컬과 불일치하나 의도는 로컬 규범과 동방향 |

행 수가 늘어 표가 다루기 어려워지면 표 대신 행별 목록(ID를 머리로 한 정의 목록)으로 써도 된다. 스키마 열이 유지되는 한 형식은 수단이다.
