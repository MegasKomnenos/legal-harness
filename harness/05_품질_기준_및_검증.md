# 품질 기준 및 검증 체크리스트

문서 생성 후 반드시 수행할 자기 검증 항목.

## 3-Layer 검증 아키텍처

검증은 3개 독립 레이어로 구성된다. Layer 1·2는 스크립트가 자동 실행하며, Layer 3은 스크립트 트리거 + 모델 재진입으로 작동한다.

| Layer | 이름 | 메커니즘 | 대상 카테고리 |
|-------|------|---------|-------------|
| **1** | 패턴 검증 | `validate_legal_doc.py` -- PostToolUse hook (Write\|Edit) | §2형식, §10, §13일부, §14, §15 |
| **2** | 인용 정합성 | `validate_citations.py` -- Stop hook (command) | §1, §2내용, §9 |
| **3** | 논증 품질 | `validate_layer3.py` -- Stop hook (command, exit 2 트리거) + 모델 재진입 검증 | §3, §5, §6, §7, §8, §11, §12, §13 |

**Layer 1 자동 탐지 항목**: em dash, 금지 표현("전혀", "불가능합니다", "결정적", "그 이유는 다음과 같습니다"), 공격적 표현, 추정적 표현("대체로", "대략"), 비표준 용어("가림 처리", "범주화", "재결례"), 편의적 번호(1), 가)), 문서 머리 콜론, 조문·판례 출처 혼동 패턴, 소극적 논증 패턴("제시하지 못하였습니다", "밝히지 않았습니다"로 끝나는 문단), 역이용 위험 패턴("이미 공개된"+"동일한 수준"), 청구 범위 자발적 제약 표현("그칩니다", "뿐입니다"), 판례 용어 선행 인용 미검증("고도의 개연성" 등을 해당 판례 인용 전에 사용).

**Layer 2 자동 탐지 항목**: 인용 사전 미수록 판례 인용([미검증 판례] 표시 없이), 법조문 인용 사전 수록 법률의 미수록 조문 인용([미검증 조문] 표시 없이), 재결 호수 형식(재결 제YYYY-NNNNN호), 판례 사건번호 형식, 교차참조 번호.

**Layer 3 스크립트 트리거 항목** (`validate_layer3.py`): (a) 소극적 논증으로 끝나는 문단(문단 단위 검사, 적극적 논증 부재 시), (b) IRAC 3단 구조 불완전(대전제만 있고 소전제/결론 누락), (c) 판례 인용 시 사실관계 비교 누락(정식 인용 후 15줄 이내). 비법률 문서 턴에서는 즉시 종료(토큰 소모 0).

**Layer 3 모델 검증 항목** (스크립트 트리거 후 모델 재진입): IRAC 3단 구조, 소극적 논증, 역이용 가능성, 판례 사실관계 비교, 부분공개 설계, 제외 선언, 선제적 방어.

## 1. 판례 인용 정확성 `[Layer 2 자동]`

- [ ] 모든 판례 인용의 사건번호·선고일·법원명이 `02_판례_인용_사전.md`와 문자 단위로 일치하는가
- [ ] 판시사항 인용 원문이 `02_판례_인용_사전.md`에 수록된 원문과 일치하는가
- [ ] `02_판례_인용_사전.md`에 수록되지 않은 판례를 새로 인용하지 않았는가 (인용 시 별도 검증 표시 필수)
- [ ] 판례 인용 형식이 "대법원 YYYY. M. D. 선고 XXXX두YYYY 판결" 형식을 준수하는가

## 2. 법조항 인용 정확성 `[Layer 1 형식 자동 + Layer 2 내용 자동]`

- [ ] 법률명이 「」로 감싸여 있는가
- [ ] 조·항·호·목·단서의 번호가 정확한가 (예: "제9조 제1항 제5호 단서", "같은 호 단서 라목")
- [ ] 약칭이 첫 등장 시 정식명칭과 함께 도입되었는가 (이하 '~'이라 합니다)
- [ ] 같은 법·같은 조·같은 항 등 약식 참조가 앞선 정식 인용과 일관되는가
- [ ] 구법 시기 판례의 판시사항에 구 조문번호가 포함된 경우, 현행법 대응 조문을 주석(※)으로 부기하였는가
- [ ] 조문의 내용을 직접 인용하거나 요약할 때 `02-1_법조문_인용_사전.md`에 수록된 원문과 일치하는가
- [ ] `02-1_법조문_인용_사전.md`에 수록되지 않은 조문을 인용하지 않았는가 (인용 시 별도 검증 표시 필수)

## 3. 논증 완결성 (IRAC) `[Layer 3 모델]`

- [ ] 각 위법사항·법리 주장이 대전제(법리)→소전제(사안 적용)→결론(위법/적법)의 3단 구조를 갖추는가
- [ ] 대전제에서 인용한 판례의 판시사항이 소전제의 사안에 적절히 포섭되는가
- [ ] 결론이 대전제와 소전제로부터 논리적으로 도출되는가
- [ ] 연관 법리 간 상호 참조가 일관되는가

## 4. 목차 구조 준수 `[Layer 3 모델]`

- [ ] 문서 유형에 맞는 `03_문서_구조_템플릿.md`의 구조를 따르는가
- [ ] 번호 체계가 일관되는가 (체계 A 또는 체계 B)
- [ ] 필수 섹션이 모두 포함되어 있는가
- [ ] 선택 섹션의 포함/제외가 사안에 적합한가

## 5. 표현 적절성 `[Layer 1 일부 자동 + Layer 3 모델]`

- [ ] `04_표현_사전.md`의 정형 표현을 적절히 사용하였는가
- [ ] 톤·레지스터가 문서 유형에 맞는가 (민원: 정중+단호, 심판: 단정)
- [ ] 감정적 표현이 없는가
- [ ] 주어가 명확한가

## 6. 제외 선언 완비 `[Layer 3 모델]`

- [ ] 공통 제외 부분이 문서 서두에 기재되어 있는가
- [ ] 각 청구 항목마다 "본 항에 대하여 명시적으로 제외하는 부분:" 이 기재되어 있는가
- [ ] 제외 범위가 해당 항목의 비공개 리스크에 적합한가
- [ ] 개인식별정보·진술 내용·사실관계 평가·토의 내용·추론 가능 세부사항이 필요에 따라 제외되어 있는가

## 7. 선제적 방어 완비 `[Layer 3 모델]`

- [ ] 예상되는 비공개 사유(제5호·제6호·제7호 등)에 대한 사전 대응이 포함되어 있는가
- [ ] 각 비공개 사유별로 "피청구기관이 ~ 원용할 가능성이 있음을 인식하고 있습니다. 그러나 ~" 패턴이 사용되었는가
- [ ] 비공개 결정 시 기재해야 할 사항 목록이 구체적으로 열거되어 있는가
- [ ] 부존재 시 안내 요청이 구체적 항목으로 열거되어 있는가

## 8. 부분공개 설계 적정성 `[Layer 3 모델]`

- [ ] 전부 비공개될 위험이 높은 항목에 대해 "비공개 대상 부분을 제외하더라도 ~ 부분은 부분공개"라는 범위별·분류별 공개 제안이 포함되어 있는가
- [ ] 부분공개의 구체적 방식(금액 구간 표시, 분류별 공개, 시행 여부의 범주, 대분류 등)이 제안되어 있는가

## 9. 교차참조 정합성 `[Layer 2 자동]`

- [ ] 별지 ↔ 법리보충 ↔ 요약본 간의 참조가 일관되는가
- [ ] "(상세 법리는 참고자료 제○항 ○목 참조)" 의 항·목 번호가 실제 참고자료와 일치하는가
- [ ] 같은 사안 내 다른 문서와의 사건번호·접수번호·일자가 일치하는가

## 10. 형식 규칙 `[Layer 1 자동]`

- [ ] 경어체가 일관되게 사용되었는가
- [ ] 번호 체계가 양쪽 괄호를 사용하는가 — `(1)`, `(가)` (편의적 `1)`, `가)` 미사용)
- [ ] 문서 머리에서 콜론(`:`) 없이 공백 2칸으로 항목-값을 분리하였는가
- [ ] 성명이 자간 벌림({{청구인_성명_자간벌림}})으로 표기되었는가
- [ ] 피청구인의 기관명과 직위가 분리({{피청구인}})되어 있는가
- [ ] 말미 서명이 "청구인  {{청구인_성명_자간벌림}}  (인)" 형식인가 — (서명 또는 인)이 아닌 (인)만
- [ ] 판례 직접 인용 시 인용부호(`"`)가 사용되었는가
- [ ] 재결 인용이 "재결 제○○호" 형식인가 — "재결례" 미사용
- [ ] 비표준 용어가 사용되지 않았는가 (→ `04_표현_사전.md` §15의 교정 목록 참조)
- [ ] 문서 제목이 사안과 문서 유형을 명확히 반영하는가

## 11. 논증 충실성 `[Layer 3 모델]`

- [ ] "~제시하지 못하였다", "~밝히지 않았다"로 끝나는 소극적 논증이 없는가 — 있다면 왜 해당 요건이 충족되지 않는지 직접 논증으로 보강되었는가
- [ ] 결론만 서술("~할 수 없다", "~에 해당하지 않는다")하지 않고, 구체적 근거(사실관계, 제외 범위, 정보의 성격 등)로 뒷받침하고 있는가
- [ ] 각 논지가 상대방에 의하여 역이용될 가능성이 없는가
- [ ] 원 청구·처분·답변서 등 기초 문서의 사실관계가 정확하게 반영되어 있는가 — 논증 편의를 위한 과소·과대 서술이 없는가
- [ ] 보충서면의 경우: 상대방의 주장을 피상적으로 넘기지 않고, 그 구체적 내용·근거를 상세히 파악한 위에서 반박이 설계되었는가

## 12. 판례 인용의 적정성 `[Layer 3 모델]`

- [ ] 판례를 인용할 때 해당 판결의 구체적 사실관계를 파악한 위에서, 이 사건과의 비교를 밝혔는가
- [ ] 사실관계가 상이하여 적용이 어려운 판례를 인용하지 않았는가
- [ ] 판례를 인용하면서 판시사항 원문을 정확히 인용하였는가 — 원문 없이 부정확한 요약만으로 인용하지 않았는가
- [ ] 이미 청구이유서 등 선행 문서에서 상세히 다룬 법리를 불필요하게 중복 인용하지 않았는가
- [ ] 보충서면의 경우: 상대방이 원용하지 않은 불리한 판례를 먼저 꺼내지 않았는가

## 13. 조문·판례 출처 구별 `[Layer 1 일부 자동 + Layer 3 모델]`

- [ ] "고도의 개연성", "비교교량", "객관적으로 현저하게" 등 판례에서 도출된 용어를 사용할 때, 해당 판례를 먼저 인용한 뒤에 사용하였는가
- [ ] 판례 용어를 사용하면서 주어를 판례로 명시하였는가 ("위 판결이 요구하는", "판례에 의하면") — "정보공개법이 요구하는 고도의 개연성" 등 조문에 귀속시키는 서술이 없는가
- [ ] 판례가 아직 인용되지 않은 맥락에서는 조문 문언("현저한 지장을 초래한다고 인정할 만한 상당한 이유" 등)을 사용하였는가
- [ ] 판례의 해석을 조문 자체의 요건·보호 범위인 것처럼 단정하지 않았는가 (예: "제5호로 보호되는 법익은 ~에 관한 것입니다" → 조문의 보호 범위는 판례 해석보다 넓을 수 있음)

## 14. AI 작문 특징 배제 `[Layer 1 자동]`

- [ ] em dash(—)를 소제목이나 본문에서 사용하지 않았는가
- [ ] "전혀", "불가능합니다", "결정적" 등 절대적·과장적 표현이 없는가
- [ ] "그 이유는 다음과 같습니다" 등 AI 도입부 패턴이 없는가
- [ ] 상대방의 능력을 깎아내리는 공격적 표현("정확히 읽지 않은 것으로 보입니다" 등)이 없는가
