# PDF 서식 사양

28개 원본 PDF 전수 분석 → 사용자가 서식을 직접 통제한 4건(청구이유서 2건, 보충서면 2건) 전 페이지 분석 → 각 요소별 최선 판단 결과.

## 확정 사양

### 용지·여백
- 용지: A4 (210 × 297mm)
- 여백: 좌 25mm, 우 25mm, 상 25mm, 하 15mm
- 본문 영역: 160 × 257mm

### 폰트
| 용도 | 폰트 | 크기 | 굵기 |
|------|------|------|------|
| 본문 | Noto Serif CJK KR | 11pt | Regular |
| 대제목 ("청 구 이 유", "보 충 서 면") | Noto Serif CJK KR | 18pt | Bold |
| 소제목 (Ⅰ. Ⅱ. Ⅲ. 수준) | Noto Serif CJK KR | 13pt | Bold |
| 중제목 (1. 가. 수준) | Noto Serif CJK KR | 11.5pt | Bold |
| 소소제목 ((1) (가) 수준) | Noto Serif CJK KR | 11pt | Regular |
| 페이지번호 | Noto Serif CJK KR | 9pt | Regular |

### 행간·단락
- 행간: 본문 크기의 2.0배 (= 22pt line height for 11pt body)
- 단락 간격: 행간의 0.5배 추가 (= 11pt after paragraph)
- 첫줄 들여쓰기: 없음 (번호 체계로 문단 구분)

### 페이지번호
- 형식: `- N -`
- 위치: 하단 중앙
- 첫 페이지에도 표시

### 문서 머리 레이아웃
```
[중앙정렬, 18pt Bold]
보 충 서 면

[좌측정렬, 11pt]
사 건  [탭→80mm] {{사건번호}} {{사건명}}
청 구 인  [탭→80mm] {{청구인_성명_자간벌림}}
피청구인  [탭→80mm] {{피청구인}}
```
- 항목명(사 건, 청 구 인, 피청구인)과 값 사이: 탭 정렬(80mm 위치)
- 성명: 자간 벌림
- 피청구인: 기관명+직위 분리

### 번호별 들여쓰기
| 수준 | 들여쓰기 | 예시 |
|------|---------|------|
| Ⅰ. / 1. | 0mm (좌측 정렬) | Ⅰ. 이 사건 통지의 처분성 |
| 1. / 가. | 0mm (좌측 정렬) | 가. 공개결정통지서의 형식 |
| (1) / (가) | 10mm | (1) 기초자료가 전자적 형태로… |
| ① | 20mm | ① 연번 |

## 제작 방식별 안내

### 방식 A: Typst (권장)
대동제-청구이유서·공사-보충서면이 Typst/LaTeX 계열로 추정됨. Typst는 한글 조판에 적합하고 CLI 기반 PDF 생성이 가능.

### 방식 B: Python (fpdf2/reportlab)
프로그래밍 기반 생성 시 fpdf2 또는 reportlab 사용. Noto Serif CJK KR 폰트 파일(.otf) 필요.

### 방식 C: HTML → PDF
HTML/CSS로 레이아웃을 구성하고 브라우저 인쇄 또는 wkhtmltopdf로 변환.

## 시스템 출력물 문서 유형 (서식 통제 불가)

다음 유형은 해당 플랫폼이 서식을 결정하므로, 하네스에서는 **텍스트 내용만 생성**하고 해당 플랫폼에 입력하여 출력한다.

| 유형 | 플랫폼 | 서식 통제 |
|------|--------|----------|
| 국민신문고 민원 | epeople.go.kr | 텍스트만 입력 |
| 정보공개청구서 | open.go.kr | 텍스트만 입력 |
| 행정심판청구서 | simpan.go.kr | 텍스트만 입력 (별지 첨부 가능) |
| 이메일 | Gmail 등 | 텍스트만 입력 |

→ 하네스가 PDF를 직접 생성해야 하는 문서 유형: **청구이유서, 보충서면, 별지, 법리보충 참고자료, 요약본** (첨부 파일로 제출하는 문서)
