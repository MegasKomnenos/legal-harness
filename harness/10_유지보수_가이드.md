# 유지보수 가이드

하네스 수정 시 정합성을 유지하기 위한 절차와 검증 방법.

---

## 1. 문제 유형 분류

하네스에 피드백을 반영하거나 항목을 추가·삭제할 때, 다음 8가지 유형의 정합성 문제가 발생할 수 있다.

| 유형 | 정의 | 발생 사례 |
|------|------|----------|
| **A. 수치 불일치** | 항목을 추가·삭제한 뒤 다른 파일의 수치를 갱신하지 않음 | README "10항목 체크리스트"(실제 14), "21개 법리"(실제 22) |
| **B. 교차참조 번호 깨짐** | 한 파일의 §번호를 변경한 뒤 이를 참조하는 다른 파일의 번호를 갱신하지 않음 | 06 제약조건 "5-1" 삽입 후 07 서두 참조 미갱신 |
| **C. 용어 불일치** | 비표준 용어를 04 §15에 등록하면서 다른 파일의 기존 사용을 교정하지 않음 | 04에서 "구간화"를 비표준으로 등록했지만 05 §8에서 그대로 사용 |
| **D. 규칙 중복** | 동일 규칙이 여러 파일에 독립적으로 기술되어 한쪽만 수정하면 불일치 발생 | 04 §17(논증 원칙) vs 07 전체 |
| **E. 규칙 모순** | 서로 다른 시점에 추가된 규칙이 충돌 | 05 §8(비표준 용어 사용) vs §10(비표준 용어 금지) |
| **F. 번호 체계 오류** | 기존 번호 사이에 비표준 번호(5-1)를 삽입하거나 물리적 순서와 번호가 불일치 | 06 "5-1", 07 §8→§7 역전 |
| **G. 정본 동기화 누락** | 정본 수정 시 사본을 함께 갱신하지 않음 | 02 판시사항 교정 시 01의 동일 원문 미갱신 |
| **H. 다중 파일 갱신 누락** | 새 항목 추가 시 관련 파일 일부를 갱신하지 않음 | 새 판례를 02에만 추가하고 01·README 미갱신 |

### 4-4. 피드백(규칙) 반영

> 아래 절차는 문서 생성 시 반복적으로 발생한 오류에 대한 교정 피드백을 하네스에 반영할 때 적용한다.

**1단계 — 정본 결정**: 이 피드백이 속하는 주제의 정본 파일을 §2 표에서 확인한다.

| 피드백 주제 | 정본 |
|-----------|------|
| 논증 방법·오류 교정 | `07_논증_가이드라인.md` |
| 금지 표현·패턴 | `04_표현_사전.md` §16 |
| 비표준 용어 | `04_표현_사전.md` §15 |
| 정형 표현 패턴 | `04_표현_사전.md` 해당 §번호 |
| 문서 구조·목차 | `03_문서_구조_템플릿.md` |
| 생성 절차·제약조건 | `06_생성_메타프롬프트.md` |
| 품질 검증 항목 | `05_품질_기준_및_검증.md` |
| PDF 서식 | `08_PDF_서식_사양.md` |

**2단계 — 충돌 확인**: 정본 파일의 기존 규칙을 읽고, 새 피드백과 충돌하는 규칙이 없는지 확인한다. 충돌이 있으면 기존 규칙을 수정하거나 새 피드백의 적용 범위를 조정한다.

**3단계 — 정본에 추가**: 정본 파일의 적절한 위치에 상세 내용을 기술한다. 새 §번호를 부여할 때는 순차적으로 부여하고, 기존 번호 사이에 "N-1" 같은 비표준 번호를 삽입하지 않는다. 번호를 재배열하면 §5의 교차참조 검증을 수행한다.

**4단계 — 참조 추가**: 다른 파일에서 이 규칙을 언급해야 할 경우, 상세 내용을 복사하지 않고 `→ {파일명} §N 참조` 형태로만 기술한다.

## 5. 수정 후 정합성 검증

하네스 파일을 수정한 후, 다음 5항목을 순서대로 확인한다.

### 5-1. 수치 검증

다음 수치가 실제와 일치하는지 확인한다.

| 수치 | 파일 | 확인 방법 |
|------|------|----------|
| 법리 수 (카테고리별 + 합계) | `README.md` 법리 범위 표 | `01`에서 법리 ID를 세어 대조 |
| 판례 수 | `README.md` 구조 표 | `02`에서 P-NNN 항목을 세어 대조 |
| 카테고리 수 | `README.md` 구조 표 | `04`에서 `## N.` 헤더를 세어 대조 |
| 체크리스트 항목 수 | `README.md` 구조 표 + `06` 기법 표 | `05`에서 `## N.` 헤더를 세어 대조 |

### 5-2. 교차참조 검증

다음 참조 번호가 실제와 일치하는지 확인한다.

| 참조 위치 | 참조 대상 | 확인 방법 |
|----------|----------|----------|
| `07` 서두 "제약조건 N~M번" | `06` 제약조건 번호 | 06의 실제 번호와 대조 |
| `06` 제약조건 11번 "§8 참조" | `07` §8 | 07의 §8 제목과 대조 |
| `07` §7 "§16 참조" | `04` §16 | 04의 §16 제목과 대조 |
| `05` §10 "§15 참조" | `04` §15 | 04의 §15 제목과 대조 |
| `03` "§14 참조" | `04` §14 | 04의 §14 제목과 대조 |

### 5-3. 용어 검증

`04_표현_사전.md` §15에 등록된 비표준 용어가 하네스 다른 파일에서 사용되지 않는지 검색한다.

```
검색 대상: harness/*.md (04 제외)
검색어: 04 §15의 비표준 열에 있는 각 용어
```

### 5-4. 정본/사본 동기화

`01_법리_데이터베이스.md`에 사본으로 수록된 판시사항 원문과 조문 원문이, 정본(`02`, `02-1`)과 일치하는지 확인한다.

- 전수 대조가 부담스러우면, 이번 수정에서 변경한 판례·조문만 대조한다.

### 5-5. 번호 순차성

각 파일에서 `## N.` 헤더를 추출하여 번호가 1부터 순차적으로 나열되는지 확인한다.

```
검색 대상: 04, 05, 06(제약조건), 07
```
