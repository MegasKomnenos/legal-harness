# 품질 기준 및 검증 체크리스트

문서 생성 후 반드시 수행할 자기 검증 항목.

## 2-Phase 검증 아키텍처

검증은 2개 Phase로 구성된다. PostToolUse hook(Write|Edit)으로 해당 파일이 수정된 직후 자동 실행된다.

| Phase | 이름 | 메커니즘 | 대상 카테고리 |
|-------|------|---------|-------------|
| **1** | 기계적 검증 | `validate_doc.py` -- PostToolUse hook (§15(a~c)는 `validate_graph.py`) | §1, §2, §10, §14, §15(a~c) |
| **2** | 내용 평가 | `validate_doc_eval.py` -> Agent 서브에이전트 | §3, §4, §5, §6, §7, §8, §9, §11, §12, §13, §15(d~e), §16, §17, §18 |

**Phase 1 자동 탐지 항목** (단일 스크립트 `validate_doc.py`로 통합 실행):

패턴 검증: em dash, AI 도입부 표현, 절대적·과장적 표현, 공격적 표현, 비표준 용어, 편의적 번호, 문서 머리 콜론. 인용 정합성: 미수록 판례/조문, 재결 호수 형식.

**Phase 2 서브에이전트 평가 항목** (Phase 1 전부 통과 후에만 트리거):

IRAC 3단 구조, 목차 구조 준수, 소극적 논증 문맥, 논증 흐름, 포섭 구체성, 청구 범위 자발적 제약, 표현 적절성, 추정적·유보적 표현, 판례 인용 깊이, 역이용 가능성, 그래프-문서 정합성, 출처 구별, 판례 용어 선행 인용, 그래프 대조, 작문 밀도 및 논증 전환, 항목별 개별 포섭, 쟁점 간 논리적 관계, 교차참조 정합성.

> `validate_doc_eval.py`는 본 체크리스트에서 `[Phase 2]`(또는 `[Phase 1 일부 + Phase 2]`) 태그가 붙은 절(§)을 직접 파싱하여 서브에이전트 평가 프롬프트를 구성한다. 따라서 Phase 2 평가 기준의 정본은 본 문서이며, 절 제목과 `[Phase N]` 태그 형식을 유지해야 파싱이 정상 동작한다.

---

## 1. 판례 인용 정확성 `[Phase 1]`

- [ ] 모든 판례 인용의 사건번호·선고일·법원명이 `data/판례_인용_사전.md`와 문자 단위로 일치하는가
- [ ] 판시사항 인용 원문이 `data/판례_인용_사전.md`에 수록된 원문과 일치하는가
- [ ] `data/판례_인용_사전.md`에 수록되지 않은 판례를 새로 인용하지 않았는가 (인용 시 별도 검증 표시 필수)
- [ ] 판례 인용 형식이 "대법원 YYYY. M. D. 선고 XXXX두YYYY 판결" 형식을 준수하는가

## 2. 법조항 인용 정확성 `[Phase 1]`

- [ ] 법률명이 「」로 감싸여 있는가
- [ ] 조·항·호·목·단서의 번호가 정확한가 (예: "제9조 제1항 제5호 단서", "같은 호 단서 라목")
- [ ] 약칭이 첫 등장 시 정식명칭과 함께 도입되었는가 (이하 '~'이라 합니다)
- [ ] 같은 법·같은 조·같은 항 등 약식 참조가 앞선 정식 인용과 일관되는가
- [ ] 구법 시기 판례의 판시사항에 구 조문번호가 포함된 경우, 현행법 대응 조문을 주석(※)으로 부기하였는가
- [ ] 조문의 내용을 직접 인용하거나 요약할 때 `data/법조문_인용_사전.md`에 수록된 원문과 일치하는가
- [ ] `data/법조문_인용_사전.md`에 수록되지 않은 조문을 인용하지 않았는가 (인용 시 별도 검증 표시 필수)

## 3. 논증 완결성 (IRAC) `[Phase 2]`

- [ ] 각 위법사항·법리 주장이 대전제(법리)->소전제(사안 적용)->결론(위법/적법)의 3단 구조를 갖추는가
- [ ] 대전제에서 인용한 판례의 판시사항이 소전제의 사안에 적절히 포섭되는가
- [ ] 결론이 대전제와 소전제로부터 논리적으로 도출되는가
- [ ] 연관 법리 간 상호 참조가 일관되는가

## 4. 목차 구조 준수 `[Phase 2]`

- [ ] 문서 유형에 맞는 `style/문서_구조_템플릿.md`의 구조를 따르는가
- [ ] 번호 체계가 일관되는가 (체계 A 또는 체계 B)
- [ ] 필수 섹션이 모두 포함되어 있는가
- [ ] 선택 섹션의 포함/제외가 사안에 적합한가

## 5. 표현 적절성 `[Phase 1 일부 + Phase 2]`

- [ ] `style/표현_사전.md`의 정형 표현을 적절히 사용하였는가
- [ ] 톤·레지스터가 문서 유형에 맞는가 (민원: 정중+단호, 심판: 단정)
- [ ] 감정적 표현이 없는가
- [ ] 주어가 명확한가

## 6. 제외 선언 완비 `[Phase 2]`

- [ ] 공통 제외 부분이 문서 서두에 기재되어 있는가
- [ ] 각 청구 항목마다 "본 항에 대하여 명시적으로 제외하는 부분:" 이 기재되어 있는가
- [ ] 제외 범위가 해당 항목의 비공개 리스크에 적합한가
- [ ] 개인식별정보·진술 내용·사실관계 평가·토의 내용·추론 가능 세부사항이 필요에 따라 제외되어 있는가

## 7. 선제적 방어 완비 `[Phase 2]`

- [ ] 예상되는 비공개 사유(제5호·제6호·제7호 등)에 대한 사전 대응이 포함되어 있는가
- [ ] 각 비공개 사유별로 "피청구기관이 ~ 원용할 가능성이 있음을 인식하고 있습니다. 그러나 ~" 패턴이 사용되었는가
- [ ] 비공개 결정 시 기재해야 할 사항 목록이 구체적으로 열거되어 있는가
- [ ] 부존재 시 안내 요청이 구체적 항목으로 열거되어 있는가

## 8. 부분공개 설계 적정성 `[Phase 2]`

- [ ] 전부 비공개될 위험이 높은 항목에 대해 "비공개 대상 부분을 제외하더라도 ~ 부분은 부분공개"라는 범위별·분류별 공개 제안이 포함되어 있는가
- [ ] 부분공개의 구체적 방식(금액 구간 표시, 분류별 공개, 시행 여부의 범주, 대분류 등)이 제안되어 있는가

## 9. 교차참조 정합성 `[Phase 2]`

- [ ] 별지 <-> 법리보충 <-> 요약본 간의 참조가 일관되는가
- [ ] "(상세 법리는 참고자료 제○항 ○목 참조)" 의 항·목 번호가 실제 참고자료와 일치하는가
- [ ] 같은 사안 내 다른 문서와의 사건번호·접수번호·일자가 일치하는가

## 10. 형식 규칙 `[Phase 1]`

- [ ] 경어체가 일관되게 사용되었는가
- [ ] 번호 체계가 양쪽 괄호를 사용하는가 -- `(1)`, `(가)` (편의적 `1)`, `가)` 미사용)
- [ ] 문서 머리에서 콜론(`:`) 없이 공백 2칸으로 항목-값을 분리하였는가
- [ ] 성명이 자간 벌림({{청구인_성명_자간벌림}})으로 표기되었는가
- [ ] 피청구인의 기관명과 직위가 분리({{피청구인}})되어 있는가
- [ ] 말미 서명이 "청구인  {{청구인_성명_자간벌림}}  (인)" 형식인가
- [ ] 판례 직접 인용 시 인용부호(`"`)가 사용되었는가
- [ ] 재결 인용이 "재결 제○○호" 형식인가
- [ ] 비표준 용어가 사용되지 않았는가 (-> `style/표현_사전.md` "비표준 용어 교정 목록" 절 참조)
- [ ] 문서 제목이 사안과 문서 유형을 명확히 반영하는가

## 11. 논증 충실성 `[Phase 2]`

- [ ] "~제시하지 못하였다", "~밝히지 않았다"로 끝나는 소극적 논증이 없는가
- [ ] 결론만 서술하지 않고, 구체적 근거(사실관계, 제외 범위, 정보의 성격 등)로 뒷받침하고 있는가
- [ ] 각 논지가 상대방에 의하여 역이용될 가능성이 없는가
- [ ] 원 청구·처분·답변서 등 기초 문서의 사실관계가 정확하게 반영되어 있는가
- [ ] 보충서면의 경우: 상대방의 주장을 피상적으로 넘기지 않고, 그 구체적 내용·근거를 상세히 파악한 위에서 반박이 설계되었는가

## 12. 판례 인용의 적정성 `[Phase 2]`

- [ ] 판례를 인용할 때 해당 판결의 구체적 사실관계를 파악한 위에서, 이 사건과의 비교를 밝혔는가
- [ ] 사실관계가 상이하여 적용이 어려운 판례를 인용하지 않았는가
- [ ] 판례를 인용하면서 판시사항 원문을 정확히 인용하였는가
- [ ] 이미 선행 문서에서 상세히 다룬 법리를 불필요하게 중복 인용하지 않았는가
- [ ] 보충서면의 경우: 상대방이 원용하지 않은 불리한 판례를 먼저 꺼내지 않았는가

## 13. 조문·판례 출처 구별 `[Phase 2]`

- [ ] "고도의 개연성", "비교교량", "객관적으로 현저하게" 등 판례에서 도출된 용어를 사용할 때, 해당 판례를 먼저 인용한 뒤에 사용하였는가
- [ ] 판례 용어를 사용하면서 주어를 판례로 명시하였는가 ("위 판결이 요구하는", "판례에 의하면")
- [ ] 판례가 아직 인용되지 않은 맥락에서는 조문 문언을 사용하였는가
- [ ] 판례의 해석을 조문 자체의 요건·보호 범위인 것처럼 단정하지 않았는가

## 14. AI 작문 특징 배제 `[Phase 1]`

- [ ] em dash를 소제목이나 본문에서 사용하지 않았는가
- [ ] "전혀", "불가능합니다", "결정적" 등 절대적·과장적 표현이 없는가
- [ ] "그 이유는 다음과 같습니다" 등 AI 도입부 패턴이 없는가
- [ ] 상대방의 능력을 깎아내리는 공격적 표현이 없는가

## 15. 그래프-문서 정합성 `[Phase 1 일부 + Phase 2]`

`validate_graph.py`가 자동 실행되어 아래 (a)~(c)를 탐지한다. (d)~(e)는 서브에이전트가 검증한다.

- [ ] (a) 그래프 JSON의 판례가 문서에 인용되어 있는가 `[Phase 1]`
- [ ] (b) 문서에 인용된 판례가 그래프 JSON에 등록되어 있는가 `[Phase 1]`
- [ ] (c) 그래프 코드가 `data/법리_데이터베이스.md`에 수록되어 있는가 `[Phase 1]`
- [ ] (d) 각 법리의 포섭(소전제)이 그래프 요약과 일관되는가 `[Phase 2]`
- [ ] (e) 법리 간 역할(주/보강/기반)이 문서의 논증 비중에 반영되었는가 `[Phase 2]`

## 16. 작문 밀도 및 논증 전환 `[Phase 2]`

- [ ] 동일 논점에 대해 유사한 법리의 판례를 과잉 나열하지 않았는가
- [ ] 판례 인용과 결론 사이에 갑작스러운 전환이 없는가
- [ ] 각 소항목의 논증이 불필요한 중복 없이 타이트하게 구성되었는가
- [ ] 문서 전체의 분량이 논점의 수와 복잡도에 비례하는가

## 17. 항목별 개별 포섭 `[Phase 2]`

- [ ] 청구 대상이 여러 항목·문서·정보로 구성된 경우, 각 항목에 대해 법리가 개별적으로 적용되었는가
- [ ] 일반론 일괄 포섭으로 그치지 않고, 항목별 개별 사정(처분 사유 적용 여부, 절차 단계, 종결·진행 여부, 분리 가능 부분 등)이 구체적으로 논증되었는가
- [ ] 상대방이 단일 사유로 일괄 처리한 항목들에 대해, 왜 일괄 처리가 부적절한지를 항목별로 개별화하여 반박하였는가

## 18. 쟁점 간 논리적 관계 `[Phase 2]`

- [ ] 주위적·예비적 관계에 있는 쟁점이 문서에서 그 관계가 명시되어 있는가 (예: "설령 ~ 하더라도")
- [ ] 하나의 쟁점이 인용되면 다른 쟁점의 전제가 달라지는 구조가 있을 때, 논리적 모순 없이 읽히는가
- [ ] 청구취지의 구제수단이 해당 절차에서 허용되는 구제범위와 정합하는가
- [ ] 해당 절차에서 고유하게 허용되는 심사범위나 판단 기준의 활용이 검토되었는가
