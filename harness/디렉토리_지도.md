# 디렉토리 문서 지도

프로젝트 디렉토리 전체의 구조와 각 문서의 분류를 정리한다.
문서 생성 시 관련 사건 폴더의 문서를 빠짐없이 참조하기 위한 용도이다.

---

## 1. 최상위 디렉토리 구조

```
legal-case-harness/
├── cases/
│   ├── 01_공사소음/   <- 사건 1: 경북대 공사 소음·비산먼지 정보공개 사건
│   ├── 02_대동제/     <- 사건 2: 대동제 행사 용역 정보공개 사건
│   └── 03_성희롱/     <- 사건 3: 컴퓨터학부 단톡방 성희롱 사건
├── 법령_판례/         <- 공통 법령·판례·내규 자료 (.txt 변환본)
│   ├── 법률_시행령/
│   ├── 판례/
│   ├── 재결례/
│   ├── 경북대_내규/
│   ├── 별표_고시/
│   ├── 기타_참고/
│   └── archive/       <- 원본 PDF 보관
├── harness/           <- 하네스 참조 파일
│   ├── data/          <- 법리·판례·조문 데이터
│   ├── style/         <- 문서 서식·표현·논증
│   ├── quality/       <- 검증·PDF 사양
│   ├── scripts/       <- 검증·출력 스크립트
│   ├── 관리_절차.md
│   └── 디렉토리_지도.md
├── legal-workbench/   <- 웹 UI (서버·클라이언트)
├── .claude/           <- Claude Code 설정 및 스킬
└── extract_all.py     <- 문서 추출 스크립트
```

> **파일 형식**: 모든 법령·판례·사건 문서는 PDF→txt 변환 완료. `.txt` 파일은 Read 도구로 직접 읽는다. 미변환 PDF가 있으면 `python extract_all.py <파일경로>`를 Bash로 실행한다.
>
> **바이너리 자산 경계**: 원본 PDF·이미지(`.pdf`/`.png`/`.jpg` 등)는 `.gitignore`로 신규 추가분이 제외되는 로컬 전용 자산이다. 저장소에는 `.txt` 변환본만 포함하는 것을 원칙으로 한다. 다만 `.gitignore` 규칙 추가 이전에 커밋된 원본 바이너리(`법령_판례/archive/` 전반과 `cases/02_대동제/archive/`·피청구인_증거의 PDF·PNG 등)는 여전히 추적된다. `.gitignore`는 이후 신규 추가분만 차단한다. clone 환경에서는 일부 원본 바이너리가 없을 수 있으므로 하네스는 `.txt`만을 정본으로 읽는다.

## 2. 사건별 폴더 상세

### 2-0. 사건 폴더 공통 구조

각 사건 폴더는 다음 하위 디렉토리로 구성된다. 사건에 따라 일부 디렉토리가 없을 수 있다.

| 하위 디렉토리 | 내용 |
|-------------|------|
| `절차_NN_차수유형_고유번호/` | 각 절차 단계를 시간·인과순으로 분리한 폴더. `절차_01_1차청구_16671281/`처럼 순번(2자리) + 차수·유형(`1차청구`·`2차청구`·`행정심판`·`민원` 등) + 고유번호(정보공개 8자리 접수번호 / 행정심판 사건번호 `YYYY-NNNNN` / 민원 신청번호)로 구성하여 청구→심판의 인과 흐름을 폴더명만으로 드러낸다. 내부 파일은 표준명(`청구서.txt`, `결정통지서.txt`, `연장통지서.txt`, `청구이유서.txt`, `답변서.txt`, `보충서면.txt`, `접수증.txt`, `접수화면.txt`, `답변서송달기안문.txt`)을 쓴다 |
| `작성_최종/` | 청구인이 확정·제출한 문서 (별지, 법리보충, 요약본, 민원 최종본 등) |
| `작성_초안/` | 작성 과정의 중간본·버전별 초안 |
| `민원/` | 본 절차의 선행이 아닌 병행 압박용 국민신문고 민원 본문 (절차 체인에 편입하지 않음) |
| `증거/` | 증거 자료 (텍스트 전사본) |
| `분석/` 또는 `법적분석/` | 법적 분석·연구·검토 문서 |
| `외부소통/` | 의원·언론 등 외부 소통 문서 |
| `피청구인_증거/` | 피청구인이 제출한 증거. 해당 심판 절차 폴더 내부에 둔다 |
| `archive/` | 하네스 생성 로그 파일 보관 (진행로그, 법리그래프, 검증이력) 및 원본 PDF |

사건 폴더 루트에는 하네스가 작업 중 생성하는 파일이 위치한다. 11단계(정리 및 보관) 완료 후 `archive/`로 이동된다:
- `진행로그_{문서명}.md`
- `법리그래프_{문서명}.md`

### 2-1. cases/01_공사소음/ — 경북대 공사 소음·비산먼지 정보공개 사건

**사건 요약**: 경북대 대구캠퍼스 내 급수탑 주변 공원 조성사업, 복지관 주변 차 없는 거리 조성사업, 지도못 환경개선공사에 관한 시설과 문서등록대장·정보목록의 정보공개를 청구하였으나, 피청구인(경북대학교)이 외형상 '공개' 결정을 하면서 실질적으로 정보를 제공하지 않고 재특정을 요구한 사건. 행정심판 진행 중.

**피청구기관**: 경북대학교 (시설과)

**사건 인과 구조**:

```

### 2-2. cases/02_대동제/ — 대동제 행사 용역 정보공개 사건

**사건 요약**: 2026년 경북대 개교 80주년 기념 대동제(5. 20.~5. 22.) 행사 용역의 발주·계약·섭외비·선정기준·정산 관련 정보공개를 청구. 1차 청구에 대한 비공개·부분공개 결정에 대해 행정심판(사건번호 2026-10093, 접수번호 2026-27402) 제기, 2차 청구에 대한 부분공개·비공개 결정에 대해 행정심판(사건번호 2026-10986) 제기. 3차 청구(접수번호 16868353)는 전부 공개 결정.

**사건 인과 구조**:

```

4. `법령_판례/판례/` 중: 정보공개 관련 대법원 판례, 성희롱·징계 관련 판례

## 5. 문서 유형별 내부 번호체계

각 사건 폴더 내 문서는 다음 패턴으로 명명되어 있다.

| 패턴 | 예시 | 설명 |
|------|------|------|
| `절차_NN_차수유형_고유번호/` | `절차_01_1차청구_16671281/`, `절차_03_행정심판_2026-10532/` | 절차 단계 폴더. 순번(2자리)+차수·유형+고유번호(정보공개 접수번호 / 행정심판 사건번호 / 민원 신청번호) |
| `{유형}_{고유번호}.{확장자}` | `청구서_16514785.txt`, `보충서면_2026-10532.txt` | **절차 폴더 내 모든 문서는 유형명 뒤에 고유번호를 붙여** 파일 단독으로 사건·절차를 구별할 수 있게 한다. 고유번호는 정보공개=접수번호, 행정심판=사건번호, 민원=신청번호 |
| `청구서_번호.txt` | | 정보공개청구서 또는 행정심판청구서 |
| `결정통지서_번호.txt` / `연장통지서_번호.txt` | | 기관의 결정·연장 통지 |
| `청구이유서_번호.txt` | | 행정심판 청구이유서 (별첨) |
| `답변서_번호.txt` | | 피청구인 답변서 (정보공개 첨부답변 / 행정심판 답변) |
| `답변서송달기안문_번호.txt` | | 답변서 송달 기안 + 위원회 송달서 |
| `보충서면_번호.txt` | | 청구인 보충서면 |
| `접수증_번호.txt` / `접수화면_번호.txt` | | 온라인 접수증·접수 완료 화면 전사 |
| `피청구인_증거/을제N호증[_주제]_번호.확장자` | `을제1호증_2026-10093.txt` | 피청구인 제출 증거 (해당 심판 절차 폴더 내부) |
| `국민신문고_기관명.txt` | 국민신문고_교육부 | 보조 `민원/` 폴더 내 기관별 병행 민원 |

| `04-1_`, `04-2_`, `04-3_` | | `작성_최종/` 내 정보공개청구 세트 하위 분류 |
