## 맥락

PDF_서식.md는 원래 "첫줄 들여쓰기 미사용이 법률서면 표준"으로 채택 0mm였다(원본 4건 중 사용은 2건뿐: 대동제-청구이유서 3.9mm, 공사-보충서면 3.5mm). 사용자 요청으로 이 표준 판단을 "사용"으로 전환했다. 1em(= 11pt ≈ 3.88mm)이 원본 사용 2건과 일치해 근거로 삼았다.

"보편적으로"의 해석을 사용자에게 확인해 적용 범위를 "일반 본문 단락만"으로 확정했다. 번호 항목((1)·①), 중앙정렬 부제, 우측정렬 꼬리는 제외.

## 터득한 것

- **번호 항목 제외 이유**: 번호 자체가 단락 시작을 표시하므로 추가 들여쓰기가 불필요하다. 특히 `.i1`/`.i2`는 이미 margin-left로 레벨 들여쓰기된 상태라, text-indent를 더하면 wrap된 둘째 줄(좌측 기준선)보다 번호 첫 줄이 더 안으로 들어가 hanging indent 역효과가 난다.
- **CSS 상속·특정성 함정**: text-indent는 상속 속성이고 `.sub`/`.i1`/`.i2`가 모두 `<p class=...>`로 렌더링되어 `p { text-indent: 1em }`에 매칭된다. 따라서 예외 4종(`.sub`, `.i1`, `.i2`, `.foot p`)에 명시적 `text-indent: 0` override가 필수다. 특정성(`.sub` 0,1,0 > `p` 0,0,1)이 override를 보장한다.
- **검증 제약**: 이 환경엔 poppler(pdftoppm)가 없어 Read PDF가 실패하고, Playwright MCP는 file: 프로토콜을 차단한다. PDF 시각 스크린샷이 막혀, `--html` 직접 출력으로 요소별 클래스·CSS를 확인하고 PDF 생성 성공(engine: playwright)으로 검증을 갈음했다.

## 작업 너머 시사점

- PDF_서식.md "판단 근거" 표는 원본 데이터가 분기(2:2)하는 요소를 사용자 선호로 재결정할 때, 채택 셀과 함께 근거 셀도 갱신해야 표와 채택이 정합한다.
- 이 환경에서 PDF 외형 변경의 시각 검증은 제약된다(poppler 부재, MCP file: 차단). HTML 구조 검증이 실질적 1차 수단이다. [[M-0006]]의 WeasyPrint→Playwright 폴백은 같은 HTML/CSS(스크립트 내 CSS 상수)를 공유하므로, 한 곳 수정이 두 엔진 모두에 반영된다.
