## 맥락

사용자의 정합성 감사(전 세션)가 `법령_판례/판례/` 두 파일의 제목 헤더가 실제 사건이 아니라 **참조판례**로 오기됨을 발견했다. `/legal-harness`로 "발견 사항 모두 중립적으로 실제 상황 파악 후 자체 개선" 지시를 받아 정정했다. 이 작업은 dream 사이클 5(종료)와 **동시 진행**되어, HEAD가 감사 시작 시 27c89e8→작업 중 37d8013(M-0017~0019·D-0008 추가)으로 움직였다([[M-0016]]·세칙 v6 '커밋 직전 git status 재확인'이 실전에서 또 유효).

## 터득한 것

- **제목 오기 계통 원인**: CaseNote/판결문 PDF 추출 시 상단 참조판례가 제목 줄로 오인 추출된다. 2011두2361→'헌법재판소 2002헌마518 결정'(참조판례), 2020두31408→'대법원 2009두19021 판결'(참조판례). 둘 다 실재하는 다른 판례라 통독 시 사건 오인 위험이 컸다.
- **self-test 사각지대 실증**: `check_case_dict_vs_files`는 파일명에서만 사건번호를 뽑아(내용 제목 미검사) 이 오류를 못 잡았다. [[D-0007]] 축B(자동검사 사각지대→통독이 메움)의 새 사례. 파일명↔내용제목 대조는 어느 CHECK에도 없다.
- **2011두2361**: 텍스트레이어는 깨끗한데 기존 교정본이 제목 오기 + 판시사항·판결요지·참조조문·주문·이유도입부를 통째 누락(이유 중간 '등 참조).'부터 시작)했다. archive 원본 재추출로 표준형식 전체 복원. 원본이 교정본보다 온전한 역전([[M-0002]] 'raw≠개선'의 반대 방향: 교정이 되레 손실).
- **2020두31408 = 벡터 PDF 강제 OCR**: 진단 결과 4쪽 모두 textlen=0·images=0·drawings 700~1100(벡터 path 웹캡처). `extract_all.py` 페이지판정("텍스트>50자면 텍스트레이어 채택")이 **부실 텍스트레이어를 골라 OCR을 건너뛰어** 깨진 추출이 저장돼 있었다. "이미지 기반이라 복원 불가"는 내 성급한 단정이었고, 사용자가 "OCR 하면 되지 않나"로 교정. 강제 렌더 OCR에서 **Tesseract psm이 결정적**: psm 3(자동)은 다단 레이아웃을 오분할해 파편화(3525자), psm 6(단일 블록)은 4846자로 이유 본문 복원. 판시부가 인용사전 P-021과 일치 확인 후 문맥·인용사전으로 교정, 익명화·불확실부는 ○○·[OCR 불확실] 표시.

## 작업 너머 시사점

- **extract_all.py 맹점(dream 후보)**: 텍스트레이어가 존재하되 부실한 벡터 path PDF는 OCR 경로로 안 넘어간다. 페이지판정에 '텍스트레이어 품질(깨짐률)' 기준을 더하거나, `관리_절차.md` 판례 변환에 '벡터 PDF는 강제 렌더 OCR·psm 6' 절차를 명문화할 여지. [[M-0006]]·[[D-0005]]의 렌더링 지식과 연결.
- **감사 주체도 편향에서 자유롭지 않다**: 나는 'OCR 불가'를 정본(진단) 없이 단정했다가 외부 관점(사용자)으로 교정됐다. [[D-0001]]·[[D-0007]]의 "편향은 자가진단 안 되고 외부가 잡는다"가 감사·개선하는 주체 자신에게도 적용된다(dream 적대검증의 인간판). 개선 작업에서도 정본(진단·재추출) 직접 대조가 먼저다.
