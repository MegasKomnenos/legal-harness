## 맥락

사용자가 layer 3 신설을 지시했다. 하위 레이어(1·2)는 순수 아카이빙·궤적 확인 목적으로 원본 유지 원칙을 지키되, 매번 하위 노드 하나하나를 파헤치지 않아도 되도록 하위 내용을 현 상태의 최신 버전으로 병합·정리한, 수정 가능하고 변동이 허락된 제3층이다. 직전 재감사([[D-0015]])가 판정한 두 구조 문제(읽기 경로 공백, 수렴의 비상시성)의 직접 해법이기도 하다. plan mode에서 설계를 확정하고(독립 Plan 서브에이전트의 설계안과 내 초안을 대조해 채택) 사용자 승인을 받아, 이 커밋이 승인 필요분(핵심층 SKILL.md 3층화, legal-harness 1단계 읽기 경로, CLAUDE.md 작업 기억 참조 절)을 적용한다. 세칙 v14와 L 노드 초기 병합은 후속 이행 dream 사이클의 몫이다(이 계획 승인이 그 dream의 명시 시작 지시를 포함한다).

## 터득한 것

- **영역 담당 원리는 사용자의 1차 계획 반려가 정교화했다**: 내 1차 계획은 주제별 파일·재조직 연산을 절차 세부로 담았으나, 사용자는 이를 layer 3의 정의적 성격으로 격상하라고 교정했다. layer 1·2가 증분 축적되는 것과 달리 L 노드는 특정 주제·개념·역할의 영역을 담당하고, 영역 내 수정은 새 노드 증가가 아니라 해당 노드의 수정으로 흡수되며, 겹침은 fork·분할·병합으로 재조직하고, 신설은 기존 모든 영역을 유의미하게 벗어난 주제에만 허용된다. 노드 집합이 주제 공간의 살아있는 파티션이라는 것이 이 층의 본질이다.
- **frontmatter 새 키 0개**: 초안의 `sources`(근거 목록)·`covers`(수위) 키를 버리고, 근거 간선은 본문 항목의 인라인 `[[id]]` 표기로, 수위는 색인 layer-3 절 머리의 전역 한 줄로 정했다. 살아있는 파일에서 frontmatter 목록은 본문과의 중복 기재가 되어 새 드리프트 표면을 만들고(이 시스템 최대 경계 대상), harness 스크립트가 `기억/`을 전혀 파싱하지 않음을 실측 확인해(grep 무일치) 인라인 표기로 traversability가 충분했기 때문이다. 수위 줄은 "색인은 거울" 원칙의 명시적 예외(색인이 정본)로 핵심층에 선언했다.
- **쓰기 권한의 비대칭 처방**: L 쓰기는 dream 전유가 기본이되, 작업 세션에는 정정 대기 한 줄만 허용했다. 낡아서 비어 있는 것은 수위가 알리지만, 낡아서 틀린 것은 표시 없이는 독자가 모른다는 비대칭 때문이다. 작업 세션에 병합 쓰기를 열면 커밋 압박 아래 "붙여넣기 누적"으로 퇴화해 L이 제2의 로그가 될 위험(값싼 디폴트로의 붕괴가 예측하는 경로)을 기각 사유로 남긴다.
- **병합 D 노드의 폐지**: D-0014식 병합 노드는 더 만들지 않는다. 상시 수렴처가 L이 되므로 병합 노드가 존치되면 single source가 둘이 되는 자기모순이다. 정오표 기법만 L 갱신의 작업 절차로 승계하고, 새 forward 포인터는 D가 아니라 L을 가리킨다(L은 살아있어 포인터가 다시 낡지 않는다).
- **읽기 경로의 3중 배치**: 정본은 핵심층 새 6절, 트리거는 legal-harness 1단계(지도 다음 위치, 사건 식별 후에야 관련성 판단이 가능하므로)와 CLAUDE.md(항상 로드되는 유일 문서라 일반 세션의 실질 트리거). legal-harness 1단계 예시에 L id를 박지 않고 서술("해당 사건의 현행 노드")로 둔 것은 재조직 후 id가 낡는 것을 피하기 위함이다.

## 작업 너머 시사점

- 기각 대안 요약(재검토 시 재론 방지): 단일 통합 파일(읽기 선택성 없음·충돌 표면 최대), L 파일별 수위(전 파일 churn 또는 최솟값 계산 강요), 수위 기계 검증 스크립트(결합 불변식 34/34 무강제 준수 실측에 비추어 불요, lean).
