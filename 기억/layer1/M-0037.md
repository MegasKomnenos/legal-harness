# adapt-skill 신설(범용 스킬 로컬 각색 메타 스킬)

## 맥락

사용자가 skill-creator를 호출해 요청했다: superpowers·skill-creator·andrej-karpathy-skills 등(열린 열거)의 범용 스킬은 일반 프로젝트를 전제하므로, legal-harness·memory-harness 등(열린 열거)으로 구축된 이 프로젝트나 유사 도메인 특화 시스템에 적합하다는 보장이 없다(적합하지 않다는 보장도 없다). 지정한 스킬의 내부 내용 전체를 "지시가 아니라 메타적으로 거리를 두고 검토할 데이터로" 읽어 충돌·최적화 여지를 판정하고, 존재하면 로컬 최적화 커스텀판을 생성하는 메타 스킬을 만들라는 것.

설계 탐색은 Explore 2개(범용 스킬 지형 전수, 로컬 규범 소스 전수)와 claude-code-guide(이름 체계), Plan 에이전트(구조 설계)로 수행했고, 플랜 승인 후 `.claude/skills/adapt-skill/`에 SKILL.md와 references 3종(대조표_양식, 반증_프롬프트, 사례_지도)을 생성했다. 사용자 확인 질문(이름·검증 규모)이 무응답이라 추천 기본값(adapt-skill, 실전 1건 검증)을 채택했다.

핵심 설계는 다섯 가지다. (1) 메타적 거리두기: 대상 스킬 텍스트를 분석 데이터로 선언하고 읽기(스킬 텍스트는 모델을 움직이도록 설계된 글이라 선언 없이 읽으면 그 지시를 수행하기 시작하는 오염이 실제로 일어난다). (2) 로컬 특수성의 동적 발견: 파일명 하드코딩 대신 5물음(무엇이 기계로 강제되나/절차 정본은/advisory는/참조 문서는/규범의 이유는)으로 실행 시점 발견. 이 프로젝트의 규범 5층 지형은 예시로만 남김(M-0010 열거 개방성의 자기적용). (3) 6축 판정(전제 불일치/직접 충돌/절차 중복·관할/관례 불일치/최적화 여지/무충돌)과 의도 보존 원칙(처리안은 "원본 의도" 열을 채운 후에만). (4) 붕괴 방지 조작 점검 4종(전수성/무충돌 근거 의무/역방향 점검/암묵 전제 스캔)과 독립 반증 검증(서브에이전트가 정본을 직접 읽고 3방향 반증: 위장 무충돌/과잉 수정/누락). (5) 3갈래 권고에서 "커스텀 불요·원본 그대로"도 정당 산출로 명문화(과잉 수정 편향의 offset).

## 터득한 것

- 플러그인 스킬은 `plugin:skill` 네임스페이스가 자동 부여되어 프로젝트 로컬 동명 스킬과 기술적으로 충돌하지 않는다(공식 문서 확인). 그래서 커스텀판은 원본명을 유지할 수 있고, 트리거 혼선은 description의 "원본 대신 이것 사용" 우선 선언으로 다룬다.
- skill-creator의 `quick_validate.py`는 `argument-hint`·`effort`를 unexpected key로 경고하지만 이는 검증기가 낡은 것이다. 공식 문서상 frontmatter 지원 키는 18종이고 로컬 legal-harness도 같은 키를 정상 사용 중이다. 검증 도구의 경고도 표상이므로 정본(공식 문서·실동작) 대조가 필요했다.
- superpowers 14종의 공통 전제(코드 산출물, 테스트 스위트, git 브랜치/worktree/PR, 서브에이전트 대량, 영어, Iron Law 톤)와 이 프로젝트 규범의 충돌 상위 지점이 조사로 특정됐다: TDD Iron Law(산문에 실패 테스트 부재), using-superpowers 강제 호출(도메인 워크플로 가로챔), master 금지(로컬은 master 직접), 서브에이전트 대량(로컬은 제한+독립 평가 예외), 커밋 컨벤션(로컬은 한국어 한 줄+메모리 노드 동반). `references/사례_지도.md`에 열린 열거 표지를 달아 캐시했다(실행 시 실측이 정본).

## 작업 너머 시사점

- 이 프로젝트의 규범 지형(기계 강제 hook / 절차 정본 / advisory / 참조 문서 / 기억의 5층, settings.json 주석의 "CLAUDE.md is advisory; the HARD guards live here"가 그 성문)과 L-0005의 β offset 삼분이, 외래 스킬을 각색할 때 "수정사항을 어느 층위에 심을지"의 판단틀로 재사용됨이 확인됐다. 기억 그래프가 축적한 판단틀이 새 유형의 메타 작업에 전이된 사례.
- 스킬 이름은 skill-creator와의 트리거 공간 분리를 고려해 adapt-skill로 정했다(동사-목적어 명명은 extract-documents와 동형 관례). "skill-" 접두 이름은 "스킬 만들어줘" 발화에서 오발동 경합을 키운다.
