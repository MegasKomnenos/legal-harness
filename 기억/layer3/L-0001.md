# 검증 인프라·self-test 지형 (현행)

이 저장소의 자동 검증이 무엇을 보고 무엇을 못 보는지의 현행 지도. 검증 인프라를 다루거나 우회·보강할 때 이 노드부터 읽는다.

## 전체 구조

검증은 3층(단계 게이트 / 문서 / 그래프) × 2단계(Phase 1 기계 검증 / Phase 2 서브에이전트 내용 평가) 격자 + 정적 self-test(`validate_harness.py`)로 구성된다. 실전 지식은 각 셀의 발화 조건과 우회 경로에 있다. 근거: [[D-0002]].

## hook 발화의 관문: 짝의 존재

step_gate·doc_eval·graph 검증은 대상 파일 인근(상위 방향만, 형제 폴더 안 봄)에서 `진행로그_*.md`·`검증이력_*.md` 짝을 찾는다. 짝이 있으면 게이트·평가하고 없으면 스킵한다. 이 단일 메커니즘이 두 작업 방식을 가른다: 로그를 archive로 옮기면 자동 스킵되어 **국한 첨삭**이 가능하고([[M-0008]]·[[M-0014]]), 로그를 사건 루트로 복원하면 게이트가 살아나 **재구성**(단계 재수행)이 강제된다([[M-0012]]). 사용자 지시의 범위 단어("국한된" 대 "전체 처음부터")가 이 분기를 가른다. 근거: [[D-0002]] 축1.

## 경로 게이트

`/harness/`·`/.claude/` 경로는 모든 게이트가 즉시 통과시킨다. 하네스·스킬 메타 개정은 진행로그 없이 자유롭다([[M-0013]]). `기억/`은 게이트 제외 목록에 없지만 진행로그 짝이 없어 실질 스킵된다(hook은 발화하되 짝 부재로 exit 0; 도입 후 전 커밋 무마찰이 방증: [[D-0015]]). 근거: [[D-0002]] 축1·축2.

## hash 재평가 스킵의 비대칭

문서는 전체 텍스트의 hash라 한 글자만 바뀌어도 재평가되고, 그래프는 JSON 블록만 hash라 mermaid·판례표·서술 메타 수정은 스킵된다([[M-0011]]). 검증이력의 위치도 다르다(문서는 별도 파일, 그래프는 파일 내부). 순수 형식 개선의 재평가 회피가 문서와 그래프에서 다르게 작동하는 근본 이유다. 근거: [[D-0002]] 축3.

## 역원용 예외의 코드 내장

validate_graph는 "문서에 인용됐으나 그래프에 없는 판례"를 원칙 실패로 보지만, 인용 앞 200자에 "피청구인…원용/주장" 패턴이 있으면 면제한다. 역원용 판례는 그래프에 등록하지 않고 문서에 쓸 수 있다. 논증 전략([[L-0002]])이 검증기 수준에서 지지되는 지점이다. 근거: [[D-0002]] 축4.

## self-test의 사정거리와 사각지대

`validate_harness.py`는 중복 기재된 식별자의 불일치(사전과 파일의 양방향 실재, 경로, 수치 쌍)만 본다. exit 0은 "화이트리스트 밖 식별자 불일치 없음"이지 의미 정합의 보증이 아니다. 실측으로 확인된 사각지대 유형: 판례 파일의 제목 헤더 오기([[M-0020]]), 서술형 사용처([[M-0007]]), git 추적 상태([[M-0021]]), 디렉토리 지도의 완전성([[M-0030]]), 인용부호 안 문구의 축자 일치([[M-0030]]), 헤더와 색인의 내적 정합([[M-0016]]·[[M-0030]]), 인용 선고일과 사전의 대조([[M-0029]]·[[M-0031]]). 이 사각지대는 통독·교차대조·사용자 감사 같은 외부 관점이 메운다. 근거: [[D-0002]] 축5, [[D-0007]] 축B.

## Phase 2는 적대적 탐색

9단계 Phase 2 서브에이전트는 체크리스트 순회 확인이 아니라 반증 태세의 적대적 탐색이며, 정보공개 거부처분 청구인 보충서면이면 논증_가이드라인 §14의 사안 적합성을 판단한다(§14 자체가 그 국면 한정이므로 다른 처분유형·서면에는 발동하지 않는다). 8단계 self-check는 기계적 선별로 한정되고 판단적 품질의 실질 게이트는 9단계다. 승격 정본: `harness/scripts/validate_doc_eval.py` 프롬프트, `legal-harness/SKILL.md` 8단계 제약·9단계. 도입 경위: [[M-0027]].

## CHECK 후보 대기 목록 (미구현, 기계화 가능 판정분)

다음은 정본이 기계 판독 가능해 self-test 편입이 가능하다고 판정됐으나 아직 미구현이다: 인용 선고일과 인용사전의 전수 대조([[M-0029]]·[[M-0031]]), 인용사전 큰따옴표 인용문과 판례 원문의 축자 대조([[M-0030]]), 디렉토리 지도와 실제 산출물의 양방향 완전성([[M-0030]]), 추적 바이너리 0건([[M-0021]]). 도입은 사용자 승인 사안이며, 기계 대조가 부적절한 항목도 있다(다른 대상을 세는 두 수치는 각주가 해법: [[M-0022]], '16-category' 동음이의 오탐: [[M-0018]]).

## 결합 불변식의 실측 현황

memory-harness 결합 불변식(작업 커밋마다 M 노드 동봉)은 도입 후 34개 커밋 전부 준수됐다(git 이력 전수 대조). 기계 강제는 불요 판정. 근거: [[D-0015]].

## 성희롱 문서군의 검증 공백 (구조적 상태)

성희롱 문서군은 진행로그·검증이력·법리그래프가 없어 hook 게이트에서 검증 대상 자체가 아니다. 선고일 오인용이 통과된 이중 사각(선고일 미대조 + 문서군 미편입)의 배경. 사건 미결 목록은 [[L-0008]]. 근거: [[M-0029]]·[[M-0031]].
